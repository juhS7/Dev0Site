🎯 Objetivo

Sair do "JS com types opcionais" e entender o sistema de tipos do TypeScript — generics, union/intersection, type guards e utility types — no nível usado em times profissionais.

1. Por que TypeScript?

TypeScript adiciona um sistema de tipos estático a JavaScript. Você escreve em .ts, o compilador checa os tipos, e o resultado é JS válido. Os ganhos práticos:

  • Erros pegos no editor antes de chegarem em produção.
  • Autocompletar inteligente em IDEs (basicamente uma documentação viva).
  • Refatorações grandes sem medo: renomear um campo quebra o build no lugar certo.

2. tsconfig.json — strict por padrão

{
  "compilerOptions": {
    "target": "ES2022",
    "module": "ESNext",
    "moduleResolution": "Bundler",
    "strict": true,
    "noUncheckedIndexedAccess": true,
    "exactOptionalPropertyTypes": true,
    "skipLibCheck": true
  }
}
⚠ Atenção

Sem "strict": true você está escrevendo "JS com decoração". Strict habilita strictNullChecks, noImplicitAny e outras checagens que são onde o TS realmente paga as contas.

3. Tipos básicos

let nome: string = "Ana";
let idade: number = 30;
let ativo: boolean = true;
let tags: string[] = ["js", "ts"];
let par: [string, number] = ["x", 1];   // tupla
let nada: null | undefined = null;
let qualquer: unknown = JSON.parse("[]"); // melhor que any

4. Types vs Interfaces

Use interface para a forma de objetos que podem ser estendidos (especialmente APIs públicas e classes). Use type para uniões, intersecções, tuplas e aliases de tipos primitivos. No dia-a-dia ambos funcionam — escolha um padrão por projeto e siga.

interface User {
  id: string;
  nome: string;
  email?: string; // opcional
}

type Status = "ativo" | "inativo" | "pendente"; // union

type AdminUser = User & { permissoes: string[] }; // intersection

5. Generics

Generics permitem escrever código reutilizável sem perder tipagem. Pense neles como "parâmetros" para tipos.

function primeiro<T>(arr: T[]): T | undefined {
  return arr[0];
}

const n = primeiro([1, 2, 3]);       // n: number | undefined
const s = primeiro(["a", "b"]);      // s: string | undefined

// Em estruturas reutilizáveis
interface ApiResponse<T> {
  data: T;
  meta: { total: number };
}

type UsersResponse = ApiResponse<User[]>;

6. Union types e type guards

Quando uma variável pode ter mais de um tipo, você precisa estreitar antes de usar. O compilador entende vários padrões.

function formata(v: string | number): string {
  if (typeof v === "string") return v.trim();
  return v.toFixed(2); // aqui v é number
}

type Cat = { tipo: "cat"; miados: number };
type Dog = { tipo: "dog"; latidos: number };
type Pet = Cat | Dog;

function descrever(p: Pet) {
  switch (p.tipo) {            // discriminated union
    case "cat": return `gato com ${p.miados} miados`;
    case "dog": return `cão com ${p.latidos} latidos`;
  }
}
💡 Dica

Discriminated unions (cada variante tem um campo literal único) é o padrão mais poderoso e idiomático de TypeScript. Use sempre que possível em vez de classes/herança.

7. Utility types essenciais

interface User { id: string; nome: string; email: string; senha: string; }

type UserPublic  = Omit<User, "senha">;            // sem senha
type UserPatch   = Partial<User>;                  // tudo opcional
type UserCreate  = Pick<User, "nome" | "email" | "senha">;
type UserMap     = Record<string, User>;           // { [id]: User }
type Required1   = Required<UserPatch>;            // tudo obrigatório

8. Decorators (visão geral)

Decorators (estágio 3, agora estáveis em TS 5) anotam classes, métodos e campos. São muito usados em frameworks como NestJS, TypeORM e Angular para metadata. Em projetos novos sem framework, prefira composição de funções — é mais simples e mais testável.

9. TypeScript no mundo real

Em projetos profissionais você raramente vai escrever any. A combinação que funciona: strict mode, schemas runtime com Zod (para validar entradas externas), discriminated unions para máquinas de estado, e generics para libs internas. Quando o tipo ficar muito complicado, é um sinal de que a modelagem dos dados pode estar errada — refatore o modelo, não o tipo.
