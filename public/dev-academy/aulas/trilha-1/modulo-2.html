2. var, let, const e escopo

var tem escopo de função e hoisting esquisito. let e const têm escopo de bloco, que é o que você espera. Use const por padrão.

function exemplo() {
  if (true) {
    var a = 1;
    let b = 2;
    const c = 3;
  }
  console.log(a); // 1  (var "vaza" do bloco)
  // console.log(b); // ReferenceError
}

3. Tipos e coerção

Primitivos: string, number, boolean, null, undefined, bigint, symbol. Tudo o resto é object (incluindo arrays e funções). Cuidado com coerção implícita: sempre prefira === a ==.

"5" == 5    // true  (coerção)
"5" === 5   // false (estrito, recomendado)
[] + []     // ""    (esquisitices)
typeof null // "object"  (bug histórico)

4. Arrays e métodos funcionais

Os três pilares funcionais que você vai usar diariamente são map, filter e reduce. Eles não mutam o array original — retornam um novo.

const users = [
  { nome: "Ana",  idade: 30, ativo: true  },
  { nome: "Bia",  idade: 25, ativo: false },
  { nome: "Caio", idade: 41, ativo: true  },
];

const nomesAtivos = users
  .filter(u => u.ativo)
  .map(u => u.nome);                 // ["Ana", "Caio"]

const somaIdades = users
  .reduce((acc, u) => acc + u.idade, 0); // 96

const ana = users.find(u => u.nome === "Ana");
💡 Dica

reduce intimida no início. Lembre: ele percorre o array carregando um acumulador. Soma, agrupamento, transformação em objeto — tudo cabe em um reduce.

5. Objetos, destructuring e spread

const user = { nome: "Ana", idade: 30, email: "ana@x.com" };

// destructuring
const { nome, ...resto } = user;       // nome="Ana", resto={idade,email}

// spread para clonar/mesclar
const atualizado = { ...user, idade: 31 };

// arrays também
const a = [1, 2, 3];
const b = [0, ...a, 4]; // [0,1,2,3,4]

6. Promises e async/await

Operações assíncronas (HTTP, leitura de arquivo, timer) retornam Promise. async/await deixa o código parecer síncrono.

async function carregarUsuario(id) {
  try {
    const res = await fetch(`/api/users/${id}`);
    if (!res.ok) throw new Error(`HTTP ${res.status}`);
    const user = await res.json();
    return user;
  } catch (err) {
    console.error("falha:", err);
    return null;
  }
}

// Paralelizar com Promise.all
const [u1, u2] = await Promise.all([
  carregarUsuario(1),
  carregarUsuario(2),
]);
⚠ Atenção

Fazer await em sequência quando podia ser paralelo é um anti-padrão clássico: dobra o tempo de resposta. Quando os awaits são independentes, embrulhe em Promise.all.

8. Tratamento de erros

Erros em JS são objetos com name, message e stack. Em projetos sérios, defina classes de erro próprias para diferenciar erros de validação, de rede, de autorização, etc.

class ValidationError extends Error {
  constructor(field, msg) {
    super(msg);
    this.name = "ValidationError";
    this.field = field;
  }
}

function parseIdade(v) {
  const n = Number(v);
  if (Number.isNaN(n) || n < 0) throw new ValidationError("idade", "inválida");
  return n;
}
