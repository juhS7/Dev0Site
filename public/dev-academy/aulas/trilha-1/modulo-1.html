🎯 Objetivo

Ao final desta aula você saberá ler, escrever e raciocinar sobre algoritmos básicos — variáveis, condicionais, loops, funções e recursão — em JavaScript. Esse é o vocabulário mínimo para todo o resto da Dev Academy.

1. O que é um algoritmo?

Um algoritmo é uma sequência finita, não ambígua e executável de instruções que, dado um conjunto de entradas, produz uma saída. Toda receita de bolo, todo cálculo de imposto e todo ranking do Spotify é, no fundo, um algoritmo. A programação é a arte de descrever algoritmos de forma que uma máquina possa executar.

Um bom algoritmo tem três propriedades: corretude (resolve o problema certo), eficiência (não desperdiça tempo/memória) e clareza (humanos conseguem ler depois). Ao longo do seu currículo de pleno, você vai trocar muitas vezes uma propriedade pelas outras — e isso é normal.

2. Variáveis e tipos de dados

Uma variável é um rótulo que aponta para um valor armazenado em memória. Em JavaScript usamos let (variável que muda) e const (referência fixa). Esqueça var — ela tem escopo confuso e foi superada.

const nome = "Maria";        // string
let idade = 29;               // number
const ativo = true;           // boolean
const tags = ["js", "ts"];    // array
const user = { nome, idade }; // object (shorthand)
let nada = null;              // ausência intencional
let indef;                    // undefined (sem valor)
💡 Dica

Use const por padrão. Só troque para let quando precisar reatribuir o valor. Isso reduz bugs e deixa a intenção explícita para quem lê.

3. Condicionais

Decisões no código são feitas com if, else if, else e switch. O resultado de uma comparação é sempre um booleano.

function classificarNota(nota) {
  if (nota >= 90) return "A";
  else if (nota >= 75) return "B";
  else if (nota >= 60) return "C";
  else return "Reprovado";
}

// switch para enumerar casos discretos
function diaDaSemana(n) {
  switch (n) {
    case 0: return "Domingo";
    case 6: return "Sábado";
    default: return "Dia útil";
  }
}

4. Loops

Loops repetem instruções enquanto uma condição for verdadeira. Os três principais:

// for clássico — quando você sabe quantas iterações
for (let i = 0; i < 5; i++) {
  console.log("rodada", i);
}

// while — quando depende de uma condição externa
let saldo = 100;
while (saldo > 0) {
  saldo -= 25;
}

// for...of — percorrer arrays e iteráveis
for (const tag of ["js", "ts", "node"]) {
  console.log(tag.toUpperCase());
}
⚠ Atenção

Loop infinito acidental é o erro nº1 do iniciante. Sempre garanta que a condição vai eventualmente se tornar falsa (atualize o contador, decremente o saldo, etc.).

5. Funções

Funções encapsulam um pedaço de lógica reaproveitável. Recebem parâmetros e devolvem um retorno. Bons nomes de função descrevem o que ela faz, não como.

// declaração tradicional
function somar(a, b) {
  return a + b;
}

// arrow function — sintaxe moderna
const dobrar = (x) => x * 2;

// parâmetros default e rest
function saudacao(nome = "anon", ...extras) {
  return `Olá, ${nome}! ${extras.join(" ")}`;
}

console.log(somar(2, 3));            // 5
console.log(dobrar(10));             // 20
console.log(saudacao("Ana", "bem-vinda")); // "Olá, Ana! bem-vinda"

6. Recursão básica

Uma função recursiva é aquela que chama a si mesma com uma entrada menor, até chegar a um caso base. É a forma mais elegante de resolver problemas que se dividem em subproblemas do mesmo tipo.

function fatorial(n) {
  if (n <= 1) return 1;        // caso base
  return n * fatorial(n - 1);  // passo recursivo
}

console.log(fatorial(5)); // 120
💡 Dica

Todo problema recursivo pode ser escrito iterativamente, mas recursão fica mais natural em estruturas em árvore (sistema de arquivos, DOM, JSON aninhado). Use a ferramenta certa para cada caso.

7. Fechando o ciclo

Você acabou de cobrir o vocabulário mínimo da programação: dados (variáveis e tipos), fluxo (condicionais e loops), abstração (funções) e autoreferência (recursão). Tudo que vem a seguir — frameworks, bancos, cloud — é construído com esses blocos.
